# Why We Model?
> [!note] We can't understand a thing, till we have modeled it.

- We build models to communicate the desires of what we want to build. To ourselves , to the people working on it in the present or the future or to a computer
- Visualize and control the system architecture
	- Basically if we give system some input we will know what it will do with data and what output it will give.
- Better understand the system we build.
- It is how the actual system will work once made.
	- It is the documentation of the actual documentation of the system.
- Exposing opportunities for simplification & reuse.
- Manage Risk
# The Importance of Modeling
- Lets say for example you are building a dog house. or some kind of building. We will need to design it around some kind of model.
- If you have the right people at the right moment then u might get a dazzling product.
- The trick is in creating the right software and in figuring out how to write less code.
- If no consideration is given to architecture , process or the tools the thing that we will build will just collapse on itself.
