# Types of Requirements
## Functional
It describes what a system does. In object oriented design *funtionality is also known as use case* 
## Non Functional
All the constrain that are on a functionality is non functionality.

## Usability
They are purely interface requirements. Like if a customer says there should be a screen for profits earned trhouhgout the year. This screen is usability. We define it to the limits of a interface. 
For example a button and how it looks and where it is is usability but what the button does is usability
